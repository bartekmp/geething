# Geething — AI Assistant Directives

## Project context

Firefox WebExtension (Manifest V3) for multi-account Gmail notifications. No backend server — all OAuth, storage, and API calls happen inside the extension.

## Architecture

```
src/background/   — service worker, OAuth, Gmail API, accounts, badge, notifications
src/popup/        — toolbar popup (account tabs, email list, inline detail)
src/options/      — settings page (opened in a tab)
src/shared/       — constants, storage abstraction, theme helpers
src/icons/        — PNG icons (generated by scripts/generate-icons.js)
tests/            — Vitest unit tests (jsdom env, browser APIs mocked)
```

## Coding conventions

- **Language**: ES2022+ with ES modules (`import`/`export`). No TypeScript, no bundler, no framework.
- **Browser API**: Always use `const api = typeof browser !== 'undefined' ? browser : globalThis.chrome;` at the module top. Never reference `browser` or `chrome` directly in logic.
- **No innerHTML with untrusted content**: Use DOM APIs (`createElement`, `textContent`, `appendChild`). Sandboxed `<iframe srcdoc>` is acceptable for rendered HTML email bodies.
- **Security**: Never log access/refresh tokens. PKCE required for all OAuth flows. Token revocation on account removal.
- **Error handling**: Validate only at system boundaries (API responses, user input). Trust internal module contracts.
- **Comments**: Only when the *why* is non-obvious. No block comments explaining what the code does.

## Testing

- Runner: Vitest with `jsdom` environment.
- Browser APIs are globally mocked in `tests/setup.js` — import from there, don't re-mock in individual tests.
- `globalThis.fetch` is a `vi.fn()` — set `mockResolvedValue` to stub HTTP calls.
- Run: `npm test` (single run), `npm run test:watch` (dev), `npm run test:coverage`.

## Commit format

```
type(scope): short description

feat(auth): add PKCE token refresh
fix(badge): clamp display at 99+
chore(ci): add web-ext lint step
```

Types: `feat`, `fix`, `refactor`, `test`, `docs`, `chore`.

## Branch naming

- `feature/<slug>`
- `fix/<slug>`
- `chore/<slug>`

## Versioning

Semver. Bump `version` in `src/manifest.json` and `package.json` together. Add a CHANGELOG entry.

## Adding a new account color

Edit `ACCOUNT_COLORS` in `src/shared/constants.js`.

## Regenerating icons

```bash
node scripts/generate-icons.js
```
